Why CSS3 Animations?

You may or may not be familiar with jQuery or canvas animation techniques. Those who use jQuery animations do so mainly for browser compatibility, and ease-of-use, but there is a better way to animating objects on screen.

jQuery and canvas utilize "path functions", where a pixel, or a DOM element is controlled pixel-by-pixel, while javascript draws its location at every point along its path.

CSS3 animations are triggered using step functions. This means that you have an initial state, and an end state, and the the browser engine determines in the best way to move objects from one state to another. Often --particularly in a desktop environment -- this means offloading the animation activity to your computer's video card, which is much better at moving stuff on screen than your CPU. This makes animations buttery smooth, and drastically lowers CPU usage on animated webpages

How to trigger?

Triggering animations is either done with pre-built interactive DOM state-change methods such as :hover, radio, and inheritence states, or using javascript to switch on and off state classes.

Triggering without javascript

The css :hover state has been around for a long time. It controls what happens when you mouse over a specified DOM element wihin the page. On the right is an example of base state and :hover state css, and animation properties which dictate the movement between the base state and hover state.

Looking at the css on the right, there are two types of properties to observe. transition-duration: 1s; and the transform: translate(0, -100px); and opacity: 0; properties. In the hover state, you will notice that the transform and opacity properties are different --these are animateable properties controlled by the animation property transition-duration.

In this article, I will only cover how to use the :hover state for firing css3 animations without javascript. For more information on other native state triggers, David Puerto is an expert.
